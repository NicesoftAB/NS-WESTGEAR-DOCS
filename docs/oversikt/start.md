## Struktur

- **Kravspecifikation**: funktionella krav uppdelade per område.
- **Användarberättelser**: implementerbara user stories med acceptanskriterier.
- **Projekt**: arkitektur, integrationer, migrering, drift, test och lansering.
- **Projektstyrning**: öppna frågor/beslut och risker som måste vara klara innan utvecklingsstart.

## Principer

- Håll kraven testbara (”det går att verifiera”).
- Beslut dokumenteras i anslutning till frågan.
- Om något är osäkert: lägg det under “Öppna frågor” direkt.
